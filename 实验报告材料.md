# CogniFlow 个性化学习路径导航系统 — 实验报告材料

---

## 一、需求分析

### 1.1 系统目标
构建一个基于 AI 的个性化学习路径导航系统，为学生提供课程学习、在线考试、智能学习计划生成三大核心功能，后台管理员可对全部数据进行增删改查管理。

### 1.2 功能需求
| 模块 | 需求描述 |
|---|---|
| 用户认证 | 注册（学生/管理员）、登录、Session会话保持、权限拦截 |
| AI智能规划 | 4种计划类型（21天挑战/速通/备考/打卡），调用豆包大模型生成Markdown计划，保存并追踪打卡进度 |
| 课程学习 | 浏览课程、查看知识点层级树、记录学习进度、按课程统计学习数据 |
| 在线考试 | 随机抽题组卷、限时答题（单选/多选/判断/简答）、自动判分、考试续接（退出可恢复）、成绩历史 |
| 后台管理 | 10个管理Tab：数据总览、课程CRUD、知识点CRUD、题库CRUD、考试记录、学生CRUD、计划监控、公告CRUD、系统日志、AI自动生成课程（含题目） |
| 系统日志 | 所有增删改操作自动记录到system_logs表 |

### 1.3 非功能需求
- 暗色/亮色双主题，所有页面一致
- BCrypt密码加密
- 数据库外键CASCADE级联删除保证数据一致性
- 前端纯原生JS+CSS，无外部UI库依赖

## 二、可行性分析

| 维度 | 分析 |
|---|---|
| 技术可行性 | Spring Boot 3 + Thymeleaf + MySQL + JPA 均为成熟技术栈，火山引擎豆包API稳定可用 |

## 三、数据库设计

### 3.1 数据库名：cogniflow，字符集utf8mb4，存储引擎InnoDB

### 3.2 表结构（12张表）

| 表名 | 主键 | 核心字段 | 外键关系 |
|---|---|---|---|
| users | id | username, password(BCrypt), email, role | — |
| courses | id | name, description, category | — |
| knowledge_points | id | name, difficulty, order_index | course_id→courses(CASCADE), parent_id→knowledge_points(SET NULL) |
| questions | id | type, title, optionA~D, answer, analysis, score, difficulty | course_id→courses(CASCADE), knowledge_point_id→knowledge_points(SET NULL) |
| study_records | id | status, score, time_spent | user_id→users(CASCADE), knowledge_point_id→knowledge_points(CASCADE) |
| study_plans | id | title, plan_type, content, total_days, progress_days | user_id→users(CASCADE) |
| chat_messages | id | role, content | user_id→users(CASCADE) |
| exam_records | id | total_score, user_score, correct_count, question_count, status, question_ids | user_id→users(CASCADE), course_id→courses(CASCADE) |
| exam_answers | id | user_answer, is_correct, score | exam_record_id→exam_records(CASCADE), question_id→questions(CASCADE) |
| announcements | id | title, content, pinned | publisher_id→users(SET NULL) |
| course_requests | id | topic, description, status | user_id→users(CASCADE) |
| system_logs | id | action, detail | user_id→users(SET NULL) |

## 六、系统流程图

### 6.1 用户登录流程
```
浏览器 → GET /login → 输入用户名密码 → POST /api/auth/login
  → UserService.login() (验密码 BCrypt) → Session.setAttribute("loginUser")
  → 返回{userId,role} → localStorage存储 → 跳转 /welcome
  → AuthInterceptor检查Session → 允许访问/拒绝(重定向/login)
```

### 6.2 AI生成学习计划流程
```
学生 → 计划页选择类型(21天/速通/备考/打卡) → 输入主题
  → POST /api/plan/generate {topic, planType}
  → PlanService 构建Prompt → ArkService.createChatCompletion(豆包大模型)
  → 返回Markdown内容 → renderMarkdown()渲染
  → 学生确认保存 → POST /api/plans {userId,title,planType,content,totalDays}
  → StudyPlanService.save() → MySQL写入study_plans表
  → 详情页 parsePlanDays()解析Day N行 → 渲染路线图 → 打卡/回退进度
```

### 6.3 在线考试流程
```
学生 → 考试页选课程 → POST /api/exam/start {userId, courseId}
  → 检查是否有IN_PROGRESS记录 → 有则恢复(resumed:true)
  → 无则随机抽10题 → examRecordRepo.save(questionIds="1,3,5,...")
  → 渲染答题界面(单选/多选/判断/简答) → 倒计时
  → 提交 POST /api/exam/submit {examRecordId, answers}
  → 逐题比对答案 → examAnswerRepo.save(每道题) → 计算总分/正确率
  → 返回{userScore, correctCount, percent, details}
  → 前端渲染成绩报告
```

### 6.4 后台AI生成课程流程
```
管理员 → 课程管理Tab → "AI生成课程"按钮 → 输入主题
  → POST /api/admin/generate-course {topic, description}
  → CourseRequestController.submit() 创建PENDING申请
  → CourseRequestController.approve() 
  → 构建JSON格式Prompt(8个知识点层级+12道题)
  → ArkService调用豆包 → 解析JSON
  → 保存Course → 保存KnowledgePoint(处理parentIdx映射)
  → 保存Questions → 返回{courseId, courseName, kpCount, questionCount}
```

## 九、总结

CogniFlow 是一个完整的 Java Web 教学管理系统，实现了需求分析中的全部功能。

**技术亮点：**
1. **AI集成** — 通过火山引擎豆包大模型实现智能计划生成和自动课程生成（含知识点树+题库），利用结构化Prompt工程生成严格JSON格式
2. **策略模式** — 学习评估采用平均分策略和加权策略可切换
3. **考试续接** — 考试中途退出可恢复，question_ids字段存储抽题记录
4. **exam_answers中间表** — 替代TEXT JSON存储答案，支持按题目维度查询正确率
5. **CASCADE级联删除** — 14条外键约束确保数据一致性
6. **系统日志** — 所有后台增删改操作自动记录
7. **全页面渐变动画** — CSS gradientFlow动画统一应用到所有页面品牌文字
8. **纯原生前端** — 无CDN依赖的markdown渲染，暗亮双主题

**文件统计：**
- Java源文件：57个（实体14 + 仓库12 + 服务7 + 控制器12 + 配置2 + 异常3 + 模式5 + 工具1 + 入口1）
- 前端模板：10个HTML
- 配置文件：pom.xml + application.properties + init.sql + start.bat
